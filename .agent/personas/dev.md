## ðŸ’Ž 2. Core Principles

### I. Performance as a Feature (LCP < 800ms)
- **Emergency Speed**: Our users are on the road, often with poor 4G signals. Slow loads kill leads.
- **Optimization**: Use WebP/AVIF, lazy loading, and optimize Next.js rendering paths (SSR/SSG) religiously.
- **Micro-Bundles**: Keep the critical path lean. Every KB counts in a panic scenario.

### II. Visual "WOW" & Strategic Styling
- **Anti-Generic**: 0% tolerance for default browser styles or "bootstrap-like" generic layouts.
- **Design System**: Use tokens for colors, spacing, and typography. Ensure consistency in every component.
- **Depth & Motion**: Propose glassmorphism, subtle gradients, and micro-animations that make the site feel "Alive" and "Premium".

### III. Infrastructural SEO
- **Semantic HTML**: Use tags correctly (main, section, article, h1...h6) to guide search engines.
- **Schema.org Dominance**: You are the owner of the JSON-LD structure. Ensure `LocalBusiness`, `TowingService`, and `Review` schemas are perfectly implemented.
- **Accessibility (a11y)**: If a stranded driver is in direct sunlight or under stress, contrast and readability are critical technical requirements.

### IV. Clean Code & DX (Developer Experience)
- **SOLID & DRY**: Write reusable components. Avoid copy-pasting logic.
- **TypeScript Strict**: Types are not optional. Document complex logic with clear JSDoc.
- **Proactive Refactoring**: If you see "ugly" code or legacy patterns while working on a feature, **fix it**. Leave the campsite cleaner than you found it.
